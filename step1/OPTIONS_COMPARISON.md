## 옵션별 기능 차이

### Option A: Storybook + CSS Variables

| 기능 | 지원 |
|------|------|
| 컴포넌트 시각적 미리보기 | ✅ 우수 (인터랙티브 UI) |
| 실시간 Props 조작 | ✅ Controls 패널로 즉시 변경 |

| 디자인 토큰 관리 | ⚠️ 수동 CSS 파일 관리 |
| 다중 플랫폼 지원 | ❌ 웹 전용 |
| 빌드 속도 | ⚠️ 중간 (Storybook 빌드 무거움) |

| 팀 협업 | ✅ 배포된 URL로 디자이너와 공유 |
| 테스트 통합 | ✅ Visual Regression Test 가능 |
| 번들 크기 | ⚠️ Storybook 자체는 무거움 (실제 배포 시 제외) |

**출력물**:
- Storybook 정적 사이트 (컴포넌트 문서)
- React 컴포넌트 라이브러리

## AI LLM 튜닝 기능 추가 제안

### 1. AI 기반 컴포넌트 생성기

```
┌─────────────────────────────────────────────────────────┐
│  💬 "파란색 Primary 버튼에 로딩 스피너 추가해줘"          │
│                                                         │
│  🤖 AI가 디자인 토큰 기반으로 컴포넌트 코드 생성         │
│                                                         │
│  [생성된 코드 미리보기]  [적용]  [수정 요청]             │
└─────────────────────────────────────────────────────────┘
```

**기능**:
- 자연어로 컴포넌트 요청
- 디자인 토큰 컨텍스트 자동 주입
- 생성된 코드 실시간 프리뷰

---

### 2. 디자인 토큰 AI 추출기

```
┌─────────────────────────────────────────────────────────┐
│  📎 Figma URL 입력: [figma.com/design/xxx...]           │
│                                                         │
│  🤖 AI가 자동으로:                                       │
│     - 컬러 팔레트 추출 및 시맨틱 네이밍                  │
│     - 타이포그래피 스케일 분석                           │
│     - 간격/반경 패턴 인식                                │
│                                                         │
│  [추출 시작]                                             │
```

**기능**:
- Figma 데이터 → Design Tokens JSON 자동 변환
- 컬러 접근성 검사 (WCAG 대비율)
- 토큰 네이밍 컨벤션 자동 적용

## 추천: Option D (하이브리드 + AI 기능)

### 구성
```
Option C (Style Dictionary)  →  토큰 관리 자동화
     +
Option A (Storybook)         →  시각적 문서화
     +
AI LLM 기능                  →  생산성 향상
```

### 기술 스택
```
figma_test/step1/
├── tokens/
│   └── design-tokens.json       # 원본 토큰 (Style Dictionary)
├── generated/
│   ├── css/variables.css        # 자동 생성
│   ├── ts/tokens.ts             # 자동 생성
│   └── tailwind.config.js       # 자동 생성
├── src/
│   ├── components/              # React 컴포넌트
│   └── ai/
│       ├── token-extractor.ts   # Figma → Token 추출
│       ├── component-gen.ts     # AI 컴포넌트 생성
│       └── sync-checker.ts      # 동기화 검사
├── .storybook/                  # Storybook 설정
└── package.json
```

### AI 기능 구현 방식
| 기능 | 구현 방법 |
|------|----------|
| 토큰 추출 | Claude API + Figma MCP 데이터 |
| 컴포넌트 생성 | Claude API + 토큰 컨텍스트 프롬프트 |
| 코드 리뷰 | Claude API + ESLint 커스텀 룰 |
| 동기화 검사 | Figma API diff + 코드 파싱 |
| 대화형 가이드 | Claude API + RAG (토큰/컴포넌트 문서) |
